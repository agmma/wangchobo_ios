# 4.2. SwiftUI View vs Swift File

> 결론: Swift File과 SwiftUI View는 같은 파일입니다.\
> 다만 SwiftUI View에는 View에 필요한 코드를 미리 입력해놓았을 뿐입니다.



### new file 화면 살펴보기

<figure><img src="../.gitbook/assets/image (61).png" alt=""><figcaption></figcaption></figure>

여기에 여러 파일들이 있는데요. SwiftUI로 코딩할 때 주로(99% 이상) 사용하는 것은 `Swift File`과 `SwiftUI View` 입니다.

(나머지는 대부분 UIKit으로 코딩할 때 필요하거나 SwiftUI와는 관련 없는 파일들입니다.)

그러면 왜 Swift File과 SwiftUI View로 나누어 있는지 궁금한데요.

파일을 생성해서 한번 알아보겠습니다.

Swift File에는 import Foundataion이외에는 입력된 코드가 없고

SwiftUI View에는 코드가 좀 입력되어 있고 오른편에 Preview도 보이네요



사실 두 파일 다 같은 Swift파일이고, 다만 SwiftUI View는 View를 만들기 위한 코드가 첨부되어 있을 뿐입니다.

```swift
//View를 위한 코드
struct SwiftUIView: View {
    var body: some View {
        Text("Hello, World!")
    }
}
```

```swift
//preview를 위한 코드
struct SwiftUIView_Previews: PreviewProvider {
    static var previews: some View {
        SwiftUIView()
    }
}
```

### Swift File은 언제 사용할까?

결론: View를 이용하지 않을 때 사용한다.



SwiftUI를 코딩하다보면 뷰 이외에도 로직에 관련된 코딩을 해야할 때가 있는데요.

예를 들어, 회원가입을 하는 화면에서 아이디나 비밀번호를 제대로 입력했는지 검증할 때는 뷰이외에도 내부적으로 아이디 형식에 맞는지, 비밀번호에 숫자와 문자가 적절히 섞였는지 검증을 해야합니다.

이때는 뷰파일과 다른 또 다른 파일에서 코딩을 해야할텐데요. 이때 Swift파일을 사용합니다.



사실 지금은 이 설명이 와닿기 쉽지 않은데요. 왜냐하면 우리는 지금까지 SwiftUI 기능만 사용해왔기 때문입니다.

Swift의 자체적인 문법은 아직 제대로 사용해본적이 없기때문에 뒤에서 Swift 문법 자체 기능을 사용한다면 자연스레 이해될 내용이기때문에 잠시 기다려주세요
